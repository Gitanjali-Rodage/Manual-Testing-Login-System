# Manual Testing – Login System

## 📌 Project Overview
This project demonstrates manual testing of a Login System.
The objective is to write and execute test cases for login functionality, password reset, and session timeout.

## 🎯 Scope of Testing
The following features were tested:
- Valid Login
- Invalid Login
- Blank Field Validation
- Password Reset Functionality
- Session Timeout

## 🧪 Testing Approach
Manual testing approach was used:
1. Created test scenarios.
2. Designed detailed test cases.
3. Executed test cases manually.
4. Logged defects in bug report file.

Both positive and negative test cases were included.

## 📂 Deliverables
- TestCases.xlsx (Test case document)
- BugReport.xlsx (Bug report document)

## 🤖 AI Integration
AI tools (ChatGPT) were used to:
- Generate structured test scenarios.
- Create edge case and negative test cases.
- Format professional bug report templates.
- Improve documentation clarity.

## 🧠 Prompt Design Approach
Prompts were carefully designed to:
- Cover positive and negative scenarios.
- Include boundary and security test cases.
- Follow industry-standard manual testing format.
- Generate structured documentation for GitHub submission.

## 🚀 How to Use
1. Open the TestCases.xlsx file.
2. Execute the test cases manually on the login system.
3. Check results and update status (Pass/Fail).
4. Refer BugReport.xlsx for logged defects.

## 📌 Conclusion
This project demonstrates understanding of manual testing concepts including:
- Test scenario design
- Test case writing
- Bug reporting
- Documentation
- Use of AI tools in software testing
